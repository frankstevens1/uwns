# AGENTS.md

## Scope

These instructions apply to `packages/lib`, the shared TypeScript library used
by apps and providers. They supplement the root repo instructions.

`packages/lib` owns shared API client contracts, DTO types, notification target
helpers, notification group metadata, and generated notification destination
metadata.

---

## Boundaries

- Keep this package framework-neutral. Do not import React, Next.js, Expo,
  React Native, or provider internals.
- Treat exported types and helpers as public shared contracts.
- Prefer explicit DTO mapping at API boundaries instead of duplicating request
  and response shapes in apps.
- Keep browser/native assumptions out of generic helpers unless the public API
  explicitly models the platform.

---

## API Client Contracts

- `createUwnsApiClient` is the shared client for authenticated API calls.
- Keep request payload mapping explicit, especially snake_case API fields versus
  camelCase TypeScript inputs.
- Preserve `UwnsApiError` status information for callers.
- Add or update exported request/response types when API endpoints change.
- Do not add retries, caching, or global side effects unless a concrete product
  requirement needs them.

---

## Notification Metadata

- Notification destinations and groups are shared product metadata consumed by
  providers, apps, and API validation.
- `notification-destinations.json` is generated by tooling. Do not hand-edit it.
- Update `tooling/notification-destination-config.json` when adding app
  destinations, then regenerate and verify the output.
- Keep destination ids stable and low-cardinality. They are stored in
  notification targets.

---

## Verification

Run relevant checks when contracts or generated metadata change:

```sh
pnpm notifications:verify
pnpm notifications:test
pnpm --filter @repo/providers check-types
pnpm --filter web check-types
pnpm --filter native check-types
```
